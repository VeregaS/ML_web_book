# ML Web Book — Образовательная платформа по Машинному Обучению

Это интерактивное веб-приложение для изучения машинного обучения. Платформа объединяет строгую математическую теорию, интерактивные визуализации и полноценную среду выполнения Python (на базе Pyodide) прямо в браузере.

---

## 🤖 РУКОВОДСТВО ДЛЯ ИИ (NotebookLM / LLM)

**Внимание ИИ-ассистенту:** Если вы читаете этот файл, ваша задача — выступать в роли методиста, архитектора учебных программ и эксперта по ML. Пользователь предоставит вам базу знаний (книги, лекции). Ваша цель — генерировать новые модули, уроки и элементы глоссария в строгом соответствии с архитектурой этой платформы.

Ниже описаны технические возможности платформы и форматы данных, которые вы должны генерировать.

### 1. Возможности платформы
* **Теория:** Поддерживает полный набор Markdown (включая таблицы).
* **Математика:** Поддерживает LaTeX через KaTeX. Используйте `$...$` для строчных формул и `$$...$$` для блочных.
* **Глоссарий (Умные подсказки):** Любое слово в теории, совпадающее с ключом из объекта `GLOBAL_GLOSSARY`, автоматически становится кликабельным. Вы **обязаны** выделять ключевые термины жирным шрифтом (например, **MSE**, **Регрессия**), чтобы привлекать к ним внимание.
* **Код (Python):** Платформа выполняет Python-код. Задания проверяются через `unittest`. Поддерживаются `numpy`, `pandas`, `scikit-learn`.
* **Тесты (Квизы):** Вопросы с выбором одного или нескольких правильных ответов.

### 2. Визуализации и Графики (ВАЖНО)
Платформа поддерживает три способа добавления графики в уроки:

#### А. Интерактивные доски (сложные визуализации)
Вы можете активировать встроенные интерактивные компоненты, указав `type` в объекте шага урока:
- `type: "svm"` — доска для демонстрации разделения классов.
- `type: "gradient_descent"` — визуализация спуска по функции потерь.

#### Б. Встраиваемые графики данных (ChartBlock)
Вы можете рисовать графики прямо в тексте теории, используя специальный блок кода `chart`. Это идеально для демонстрации трендов, сравнения метрик или распределений.
**Пример использования в теории:**
```chart
{
  "type": "line", // или "scatter"
  "title": "Сравнение MSE и MAE",
  "xLabel": "Отклонение",
  "yLabel": "Значение ошибки",
  "data": [
    {"x": 0, "y": 0},
    {"x": 1, "y": 1},
    {"x": 2, "y": 4}
  ]
}
```

#### В. Python Графики
В заданиях на код (type: "code") пользователь может использовать `matplotlib.pyplot`. Графики будут автоматически отрисованы в окне вывода.

### 3. Формат Глоссария (`src/utils/glossary.js`)
Определения должны быть строгими, содержать формулы (LaTeX) и таблицы.

```javascript
export const GLOBAL_GLOSSARY = {
  'Термин': {
    definition: `**Термин** — определение с формулой $$E=mc^2$$`,
    example: `Пример или блок кода \`\`\`python ... \`\`\``
  }
};
```

### 4. Формат Уроков (`src/content/lessons.js`)
Вся программа — это массив `lessons`. Каждый шаг содержит теорию и задание.

```javascript
export const lessons = [
  {
    id: "lesson-id",
    module: "Глава",
    title: "Урок X",
    steps: [
      {
        id: "step-1",
        // type: "svm" // Опционально: включить доску
        theory: "Текст с **Жирными Терминами** и графиком ```chart ... ```",
        task: { type: "code", content: "..." },
        initialCode: "...",
        testCode: "import unittest..."
      }
    ]
  }
];
```
