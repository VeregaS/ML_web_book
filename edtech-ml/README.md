# ML Web Book — Образовательная платформа по Машинному Обучению

Это интерактивное веб-приложение для изучения машинного обучения. Платформа объединяет строгую математическую теорию, интерактивные визуализации и полноценную среду выполнения Python (на базе Pyodide) прямо в браузере.

---

## 🤖 РУКОВОДСТВО ДЛЯ ИИ (NotebookLM / LLM)

**Внимание ИИ-ассистенту:** Если вы читаете этот файл, ваша задача — выступать в роли методиста, архитектора учебных программ и эксперта по ML. Пользователь предоставит вам базу знаний (книги, лекции). Ваша цель — генерировать новые модули, уроки и элементы глоссария в строгом соответствии с архитектурой этой платформы.

Ниже описаны технические возможности платформы и форматы данных, которые вы должны генерировать.

### 1. Возможности платформы
* **Теория:** Поддерживает полный набор Markdown (включая таблицы).
* **Математика:** Поддерживает LaTeX через KaTeX. Используйте `$...$` для строчных формул и `$$...$$` для блочных.
* **Глоссарий (Умные подсказки):** Любое слово в теории, совпадающее с ключом из объекта `GLOBAL_GLOSSARY`, автоматически становится кликабельным. По клику открывается окно с определением. Вы **обязаны** выделять ключевые термины жирным шрифтом (например, **MSE**, **Регрессия**), чтобы привлекать к ним внимание, если они есть в глоссарии.
* **Код (Python):** Платформа выполняет Python-код в браузере. Задания на код проверяются через библиотеку `unittest`. Поддерживаются `numpy`, `pandas`, `scikit-learn`.
* **Тесты (Квизы):** Платформа поддерживает вопросы с выбором одного или нескольких правильных ответов.
* **Интерактивные доски:** В платформу встроены готовые визуализации. Вы можете активировать их, указав `type: "svm"` или `type: "gradient_descent"` в шаге урока.

### 2. Формат Глоссария (`src/utils/glossary.js`)

Каждый новый сложный термин, вводимый в курсе, должен быть добавлен в глоссарий.
Определения должны быть строгими, содержать формулы (LaTeX) и таблицы, если уместно. В поле `example` нужно давать понятные аналогии или примеры кода.

**Схема генерации термина:**
```javascript
export const GLOBAL_GLOSSARY = {
  // Ключ - точное совпадение слова или фразы (регистр не важен)
  'Название термина': {
    definition: `**Название термина** — это строгое определение. Можно использовать формулы:
$$ \sum_{i=1}^{n} x_i $$
Включайте Markdown таблицы и списки для структурирования.`,
    example: `Здесь приводите понятную аналогию из жизни или пример кода:
\`\`\`python
import numpy as np
\`\`\``
  }
};
```

### 3. Формат Уроков (`src/content/lessons.js`)

Вся учебная программа хранится в массиве `lessons`. Каждый урок делится на шаги (`steps`). Шаг содержит теорию и обязательное практическое задание (код или тест).

**Схема генерации урока:**
```javascript
export const lessons = [
  {
    id: "уникальный-id-урока",
    module: "Название Главы (Например: Нейронные сети)",
    title: "Урок X: Название урока",
    difficulty: "Легко" | "Средне" | "Сложно",
    description: "Краткое описание того, что будет в уроке (1-2 предложения).",
    steps: [
      {
        id: "step-1",
        // type: "svm" | "gradient_descent" // (Опционально) Включает интерактивную доску
        theory: `
Текст теории в формате Markdown. 
Обязательно выделяйте термины жирным: **Градиентный спуск**, **Веса**.
Добавляйте математику: $y = wx + b$.
Можно использовать таблицы.
        `,
        // ЗАДАНИЕ: ТИП "КОД"
        task: {
          type: "code",
          content: "Текст задания. Что нужно написать в редакторе."
        },
        hint: "Подсказка для пользователя, если он застрял.",
        initialCode: "def my_func():\n    # Напишите код\n    pass",
        testCode: `
import unittest
import numpy as np
import student_code

class TestTask(unittest.TestCase):
    def test_logic(self):
        res = student_code.my_func()
        self.assertEqual(res, expected_value, "Сообщение об ошибке")
`
      },
      {
        id: "step-2",
        theory: `Продолжение теории...`,
        // ЗАДАНИЕ: ТИП "КВИЗ"
        task: {
          type: "quiz",
          questions: [
            {
              question: "В чем суть регуляризации?",
              multipleChoice: false, // true если ответов несколько
              options: [
                { id: "1", text: "Уменьшение весов для предотвращения переобучения" },
                { id: "2", text: "Увеличение скорости обучения" },
                { id: "3", text: "Удаление выбросов" }
              ],
              correctAnswers: ["1"]
            }
          ]
        },
        hint: "Вспомните формулу штрафа за большие веса."
      }
    ]
  }
];
```

### 4. Инструкция по генерации контента для NotebookLM

Если пользователь просит вас **создать новый курс / главу / урок**, действуйте по следующему алгоритму:

1. **Анализ источников:** Изучите базу знаний пользователя (загруженные книги и лекции).
2. **Проектирование плана:** Предложите пользователю структуру модуля (список уроков и шагов внутри них).
3. **Генерация Глоссария:** Составьте JSON-объект с новыми терминами, которые встретятся в предложенном плане. Убедитесь, что определения строгие, математически точные и содержат глубокие примеры (аналогии, код, таблицы).
4. **Генерация Уроков:** Выдайте готовый JS-код для массива `lessons`.
   * Пишите глубокую, университетского уровня теорию.
   * Обязательно вставляйте формулы LaTeX (`$$...$$`).
   * Выделяйте ключевые слова `**жирным шрифтом**`, чтобы движок платформы мог превратить их в интерактивные подсказки глоссария.
   * Для заданий на программирование (type: "code") пишите надежные тесты на базе `unittest` в поле `testCode`. Изолируйте тесты так, чтобы они проверяли логику, а не просто синтаксис.
   * Чередуйте написание кода с тестами-квизами (type: "quiz") для проверки концептуального понимания.
